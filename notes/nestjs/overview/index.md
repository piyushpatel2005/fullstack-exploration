# Overview of Nest.js

NestJs is a progressive Node.js framework which includes elements of OOP (Object Oriented Programming), Functional Programming and Reactive programming to build efficient and scalable server-side applications. It uses Typescript to provide a strong typing system and is highly modular to build large scale applications.

## History of Nest.js

NestJS is inspirted by Angular and is built on the foundations of Typescript. It was developed by Kamil Mysliwiec and is maintained by the NestJS community. It was first released in 2017 and has since been adopted by many developers. By leveraging type safety of Typescript, it allows developers to write robust, maintainable and error-free code. NestJS is built on top of Express.js which is a popular web framework for Node.js. However, NestJS adds a lot of additional features to Express.js like dependency injection, routing and middleware, making it a full-fledged framework for building server-side applications.

In Javascript, there are several backend frameworks such as Express.js, Koa, Sails.js, Fastify, Nest.js, etc. Each of them have their own design choices and features. The choice of framework depends on several factors such as size and complexity of the project, team experience and also the requirements of the application.

## Features of NestJS

Node.js is a popular framework for building scalable and efficient web applications but it's difficult to develop large and complex applications. NestJS makes it easier to develop large application by providing below features.

- **Dependency Injection:** NestJS provides dependency injection which easily allows you to inject dependencies into your controllers and services. This also makes your code more modular and testable.
- **Routing:** NestJS provides built-in routing which allows you to define routes for your application. This also makes your code more modular and testable.
- **Middleware:** Middleware is the code that runs between the request and the response cycle. NestJS provides several middleware functionalities to your application such as logging, authentication, error handling etc.
- **Validation:** NestJS provides built-in validation which allows you to validate your data before it is passed to your application. This also makes your code more modular and testable.
- **Testing:** NestJS provides built-in testing with mocking capabilities which allows you to test and validate your business logic.
- **Reactive Programming:** NestJS provides reactive programming which allows you to build event-driven applications.

### Why should you use Nest.js?

It's essential to know the features but why should you use Nest.js and not any other framework for your application?

- NestJS uses convention over configuration approach which means many of the settings are already configured with default values but can be overridden by the user.
- NestJS is highly scalable and can handle large number of requests.
- NestJS is modular and therefore easy to maintain in the long term for complex applications.
- With the support of Typescript, NestJS provides strong typing system which makes your code less error-prone. If you're new to Typescript, please check out the [Typescript tutorial](../../typescript/).
